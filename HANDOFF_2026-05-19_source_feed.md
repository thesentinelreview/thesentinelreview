## What This Adds to the Picture

As of May 19, 2026, the Conflict Intelligence Dashboard now has two
user-facing modes:

1. **AI synthesis** at `/app` — the existing analyst dashboard (map, alerts,
   intensity, briefing, top sources). Sign-in required, Analyst tier or
   above.
2. **Source Feed** at `/app/feed` — chronological list of the raw OSINT
   posts that informed published events, English-translated where needed,
   filterable by theater / platform / trust tier. **Free, no sign-in
   required.** This is the "AI toggle" called out as Queued in v2.

A separate `/theaters` index plus `/theaters/[theater]` detail pages are
the SEO + funnel surface for each of the four theaters (also Queued in v2).

The product position from the May 18 strategy thread stands: **AI synthesis
is the eventual paid lever**, Source Feed is the free top of funnel. No
paywall scaffolding is in place yet — both modes are free in beta.

## New Data Model Surface

### `raw_posts.translated_text` (column)

Added by migration 0008. NULL means one of:

- Source is already English (`lang = 'en'`)
- Pre-filter skipped: link-only post, empty text
- Translator failed (JSON parse, API error)
- Row predates the translator and has not yet been backfilled

Non-NULL means: English translation generated by `claude-haiku-4-5`. The
original text in `raw_posts.text` is **always** preserved untouched.

### `llm_logs` entries with `purpose = 'translate_raw_post'`

One row per translator API call (live ingest **and** backfill). Backfill
rows are distinguishable by `job_id IS NULL`. The `prompt` and `response`
columns hold the user message and the raw JSON response — useful for
debugging bad translations after the fact.

## Environment & Secrets

No new required secrets. The translator and backfill reuse the existing
`DATABASE_URL` and `ANTHROPIC_API_KEY`.

Optional overrides:

| Variable                              | Default              | Where it matters             |
|---------------------------------------|----------------------|------------------------------|
| `ANTHROPIC_MODEL_TRANSLATE`           | `claude-haiku-4-5`   | Translator model selection   |
| `SENTINEL_BACKFILL_DAYS`              | `30`                 | Backfill window              |
| `SENTINEL_BACKFILL_RATE_LIMIT`        | `5.0`                | Backfill max calls/sec       |
| `SENTINEL_BACKFILL_BATCH_SIZE`        | `100`                | Backfill DB page size        |
| `SENTINEL_BACKFILL_MAX_POSTS`         | `0` (no cap)         | Backfill safety cap          |

The backfill workflow exposes `days`, `rate_limit`, and `max_posts` as
`workflow_dispatch` inputs — set them in the GitHub Actions UI when
triggering manually.

## Decisions Worth Remembering

- **Translation lives on the live ingest path.** When `extract_events`
  runs, every post that's not pre-filtered as English goes through the
  translator before the extractor sees it. Failures are non-blocking —
  a NULL `translated_text` simply renders the original on the feed with
  a "Translation unavailable" badge.
- **Pre-filter runs before any API call.** The order is:
  1. empty / whitespace → `skipped=empty`
  2. link-only (after URL stripping, <5 chars) → `skipped=link_only`
  3. ASCII-ratio ≥ 0.95 AND zero non-Latin chars → `skipped=english_heuristic`
  4. otherwise → API call
  This catches the cheap cases without burning API budget.
- **The Source Feed only shows posts that have been linked to a published
  event.** Posts that haven't been extracted into an event yet (or were
  skipped as `no_event_signal`) don't appear. This is intentional — those
  are the noisy long tail. If we want to surface them later, the query in
  `lib/queries.ts` is the one place to change.
- **`/app/feed` is exempt from the Clerk auth gate.** Done via a parallel
  `isPublic` route matcher in `apps/web/proxy.ts`. The rest of `/app/`
  stays sign-in-required. The mode toggle from feed → AI synthesis will
  redirect a Watch user to sign-in (correct: AI synthesis is sign-in-gated
  today).
- **Theater landing-page copy lives in `apps/web/data/theaters.ts`.** Edit
  prose, taglines, key actors, and SEO meta there. Nothing else needs to
  change.

## Known Limitations & Deferred Work

1. **Paywall scaffolding for AI synthesis is not built.** Both modes are
   free in beta. The plan from May 18 is: validate the toggle, then put
   AI synthesis behind Analyst tier. The Clerk + Stripe plumbing is
   already in place; the wiring is the small work that's left when
   product is ready.
2. **Backfill window is capped at 30 days by default.** Older history can
   be translated by re-running with a larger `days` input, but the spec
   for "is this worth the API spend on year-old posts that no one will
   read?" is unanswered.
3. **No translator-quality monitoring.** We log every call to `llm_logs`
   but there's no dashboard surfacing translation-failure rate, length-
   ratio outliers, or per-source skip rates. A future PR could
   add a check to `apps/ingest/sentinel/checks.py`.
4. **`apps/web/AGENTS.md` warns "This is NOT the Next.js you know".** The
   project is on Next 16.2.6 with the new async-`searchParams` /
   `proxy.ts` conventions. If you make frontend changes, read
   `node_modules/next/dist/docs/` before writing code. Clerk 7.x in this
   version does **not** export `<SignedIn>` / `<SignedOut>` from
   `@clerk/nextjs` — use server-side `auth().userId` for conditional
   rendering, as `/app/feed` does.
5. **Pre-existing `held_for_review = true` rows.** The autonomy decision
   from May 18 means new events never set this. Old held rows still exist
   in the database; the `held_events` integrity warning will fire for them
   indefinitely unless one-off SQL clears the flag. Acceptable.
6. **Supabase migration was discussed and parked.** No code change. If the
   Claude mobile app's native Supabase connector becomes a meaningful
   workflow win, the migration is non-trivial but tractable.
