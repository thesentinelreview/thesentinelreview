# Handoff — Login + Paywall Implementation (2026-05-16)

## What Was Built

Added Clerk authentication and a Stripe paywall to the Sentinel Intelligence Dashboard.

- **`/`** (Watch tier) — remains fully public. All 4 theaters accessible. 30d window locked. Full briefing archive locked. Upgrade nudge bar at bottom.
- **`/app`** (Analyst tier) — protected. Redirects unauthenticated users to `/sign-in`. After sign-in + paid subscription, shows full dashboard: all theaters, all windows (24h/7d/30d), full briefing history, `<UserButton />` in topbar.
- **`/sign-in`** and **`/sign-up`** — Clerk-hosted auth pages, dark background.
- **`/api/checkout`** — POST, creates Stripe Checkout session for Analyst subscription.
- **`/api/webhooks/stripe`** — receives `checkout.session.completed`, `customer.subscription.updated`, `customer.subscription.deleted`; syncs into `user_subscriptions` table.

## Architecture

| Layer | Choice |
|-------|--------|
| Auth | Clerk v7 (`@clerk/nextjs`) |
| Middleware | `apps/web/proxy.ts` (Next.js 16 renamed `middleware.ts` → `proxy.ts`) |
| Subscription state | Neon Postgres `user_subscriptions` table |
| Payments | Stripe v22, Checkout Sessions |
| Tier check | `apps/web/lib/auth.ts` → `getUserTier()` |

## How to Test End-to-End

1. Open `dashboard.thesentinelreview.com/app` in incognito → lands on sign-in ✓
2. Sign up with a test email
3. After sign-up you'll be on `/app` but with Watch tier (no `user_subscriptions` row yet) → should redirect to `/pricing`
4. Click "Claim Founding Analyst" on the pricing page → Stripe Checkout
5. Use test card `4242 4242 4242 4242`, any future expiry, any CVC
6. Checkout success → redirects to `/app?checkout=success`
7. Stripe webhook fires → `user_subscriptions` row created with `tier='analyst'`, `status='active'`
8. `/app` loads the full Analyst dashboard
